## 1. 프로젝트 개요

### 1.1 목적

**"부품을 지금 사야 할까? 나중에 사야 할까?"**

GPU(그래픽카드)를 구매할 때 최적의 타이밍을 예측하는 AI 시스템입니다.
바둑에서 AlphaGo가 승률을 계산하듯이, 이 시스템은 **구매 적정도(0~100점)**를 계산합니다.

### 1.2 핵심 개념

```
입력:  GPU 모델명 (예: RTX 5060)
처리:  MCTS 50회 시뮬레이션 → 4개 신호 혼합 정책 보정
출력:  BUY_NOW / WAIT_SHORT / WAIT_LONG / HOLD / SKIP + 신뢰도
```

### 1.3 바둑 승률과의 비교

| 바둑 (AlphaGo) | GPU 구매 AI |
|----------------|-------------|
| 현재 국면 분석 | 현재 가격 + 시장 분석 |
| 여러 수 시뮬레이션 | 여러 구매 시점 시뮬레이션 |
| 승률 70% | 구매 적정도 70% |
| "여기에 두세요" | "지금 사세요 / N주 대기" |

### 1.4 왜 AlphaZero인가?

단순 가격 예측이 아닌 **전략적 판단**이 필요하기 때문입니다.

- ❌ 단순 회귀: "다음 주 가격은 60만원"
- ✅ AlphaZero: "1주일 대기하면 +5% 이득, 지금 사면 +2% 이득 → 대기 추천"

## 5. AlphaZero AI 엔진

### 5.1 아키텍처 개요

```
[입력] GPU 모델명 "RTX 5060"
  ↓ resolve + state vector 로드
[256D 상태 벡터]
  ↓
[h] Representation Network
  • Linear(256, 256) + LayerNorm
  • FeedForward×3 (잔차 연결: x = x + ff(x))
  • 각 블록: 256→512(GELU)→256
  ↓ 256D Latent State
[f] Prediction Network → Policy logits(5D) + Value(1D)
  ↓
[MCTS] 50회 시뮬레이션
  • PUCT: Q + c·P·√N_parent / (1 + N_child), c=√2
  • 루트 Dirichlet noise: ε=0.25, α=0.03
  • 5-step rollout: g 호출
  ↓
[g] Dynamics Network (rollout 중 호출)
  • 입력: [latent(256D); action(5D)]
  • 4개 잔차 블록 (256→512(GELU)→256)
  • 출력: next_latent(256D), reward_μ, reward_log σ²
  ↓
[a] Action Model (policy calibration)
  • ActionPriorNetwork: 256→128→64→5
  ↓
[정책 보정]
  • 45%·MCTS + 25%·Reward + 15%·f-net prior + 15%·ActionModel prior
  • Anti-collapse: entropy < 0.65 시 uniform(1/5) 혼합
  ↓
[안전 게이트]
  • confidence < 0.25 → HOLD
  • entropy > 1.58 → HOLD
  ↓
[최종 결정] BUY_NOW / WAIT_SHORT / WAIT_LONG / HOLD / SKIP
```

### 5.2 네트워크 상세

#### (1) Representation Network h(s)

```python
class RepresentationNetwork(nn.Module):
    # 입력: 256D GPU 시장 상태 벡터 (feature_engineer 출력)
    # 출력: 256D Latent State

    def forward(self, state_tensor):
        x = self.input_embedding(x)   # Linear(256, 256) + LayerNorm
        # 잔차 연결: 그래디언트 소실 방지
        x = x + self.ff1(x)
        x = x + self.ff2(x)
        x = x + self.ff3(x)
        x = self.layer_norm2(x)
        return self.output_layer(x)   # (batch, 256)

    # ※ PositionalEncoding: seq_len=1에서 의미 없으므로 forward에서 적용 안 함
    #   (체크포인트 호환성을 위해 __init__에만 유지)
```

#### (2) Dynamics Network g(s, a)

```python
class DynamicsNetwork(nn.Module):
    # 입력: latent(256D) + action one-hot(5D)
    # 출력: next_latent(256D), reward_mean, reward_logvar(=log σ²)

    def forward(self, s_t, a_t):
        x = torch.cat([s_t, a_t], dim=-1)  # (batch, 261)
        x = self.input_layer(x)
        x = self.layer_norm1(x)
        for block in self.blocks:
            x = x + block(x)               # 잔차 연결
        x = self.layer_norm2(x)
        s_tp1 = self.next_state_head(x)
        reward_mean = self.reward_mean_head(x).squeeze(-1)
        reward_logvar = self.reward_logvar_head(x).squeeze(-1)  # raw, 활성화 없음
        return s_tp1, reward_mean, reward_logvar

    # σ² = exp(reward_logvar)  — Gaussian NLL로 불확실성 헤드 학습
```

#### (3) Prediction Network f(s)

```python
class PredictionNetwork(nn.Module):
    # 입력: latent(256D)
    # 출력: policy_logits(5D), value(1D, Tanh 범위 [-1, 1])

    def forward(self, latent):
        x = self.input_layer(latent)
        for block in self.blocks:
            x = x + block(x)           # 잔차 연결
        x = self.layer_norm2(x)
        policy_logits = self.policy_head(x)   # 512→2048→5
        value = torch.tanh(self.value_head(x)) # 512→512→1
        return policy_logits, value
```

#### (4) Action Model a(z)

```python
class ActionModel(nn.Module):
    # 두 가지 역할:
    # 1. ActionEmbeddingLayer: action index(int) → 16D embedding
    # 2. ActionPriorNetwork: latent(256D) → action prior probs(5D)
    #
    # gpu_purchase_agent.py 의 하드코딩 휴리스틱(utility_bias)을 대체

    def get_prior(self, latent_state):
        # latent_state: (batch, 256) or (256,) — 둘 다 지원
        probs = self.prior_net.prior_probs(latent_state)
        return probs  # 합이 1인 확률 분포
```

### 5.3 MCTS (AlphaZero 스타일)

```
PUCT 선택 기준:
  UCB(s,a) = Q(s,a) + c · P(s,a) · √N(s) / (1 + N(s,a))
  c = √2 ≈ 1.4142

루트 노드 탐색 다양성 (Dirichlet noise):
  P'(s,a) = (1 - ε) · P(s,a) + ε · Dir(α)
  ε=0.25, α=0.03

롤아웃 (5 steps):
  각 스텝: f(z)로 행동 샘플링 → g(z, a)로 다음 상태·보상 예측
  누적 보상: Σ γ^k · r_k + γ^5 · V(z_5), γ=0.99
```

### 5.4 체크포인트 구조

```python
checkpoint = {
    "h_state_dict": ...,   # Representation Network 가중치
    "g_state_dict": ...,   # Dynamics Network 가중치
    "f_state_dict": ...,   # Prediction Network 가중치
    "a_state_dict": ...,   # Action Model 가중치 (없으면 하드코딩 휴리스틱 폴백)
    "meta": {
        "action_labels": {0:"BUY_NOW", 1:"WAIT_SHORT", ...},
        "num_samples": 620,
        "action_prior": [0.15, 0.25, 0.35, 0.20, 0.05],  # 경험적 행동 분포
        "train_config": {"num_steps": 500, "batch_size": 32, "lr": 1e-4},
        "schema_version": "agent-v1",
    }
}
```

## 6. 학습 프로세스

### 6.1 Transition 데이터셋 구성

학습은 연속된 날짜 쌍에서 전이 샘플을 생성합니다:

```python
# (state_t, state_{t+1}, action, reward, value_target)
pct_change = (price_{t+1} - price_t) / price_t

# 행동 레이블링 (평가기 보상 함수와 일치):
if pct_change >= +0.02:  action = BUY_NOW    # 내일 비싸지므로 오늘 구매
elif pct_change <= -0.02: action = WAIT_LONG  # 크게 하락 예정 → 기다려라
elif pct_change <= -0.005: action = WAIT_SHORT
else:                     action = HOLD

reward = clip(pct_change, -1.0, 1.0)
value_target = tanh(pct_change * 8.0)
```

> **설계 원칙**: SKIP(4) 레이블은 가격 변화율로 정할 수 없음
> (가격 하락 시 WAIT가 최적이며, SKIP의 평가 보상은 항상 음수)

### 6.2 Loss 함수 (5개 + 정규화)

```python
total_loss = (
    policy_loss              # 교차 엔트로피 (클래스 가중치 적용)
    + value_loss             # MSE (value 예측)
    + dynamics_loss          # MSE (next state 예측)
    + reward_loss            # MSE (reward mean 예측)
    + 0.5 * reward_nll_loss  # Gaussian NLL: logvar 헤드 불확실성 학습
    - 0.001 * entropy        # 엔트로피 정규화 (다양성 장려)
    + 0.02 * prior_reg       # KL(policy ‖ empirical prior) (분포 편향 방지)
    + 0.3 * action_prior_loss  # ActionModel 지도 학습 (행동 prior 정확도)
)

# Gaussian NLL (logvar 헤드만 업데이트):
reward_nll_loss = 0.5 * (
    (reward_pred.detach() - rewards)² * exp(-reward_logvar) + reward_logvar
).mean()
```

## 11. FAQ

### Q1. 이 시스템은 진짜 AI인가요?

**A:** 예! AlphaGo와 동일한 원리의 AlphaZero 아키텍처에 추가 개선이 포함됩니다.

- ✅ 4개 신경망 (h/g/f/a, ~19M 파라미터)
- ✅ MCTS 탐색 (PUCT 공식, Dirichlet noise)
- ✅ 5개 손실 함수 기반 Fine-tuning
- ✅ 자동 학습 + 릴리즈 파이프라인

### Q2. 지금 당장 사용할 수 있나요?

**A:** 현재 체크포인트(`alphazero_model.pth`)가 있으면 즉시 사용 가능합니다.
30일 실데이터 학습 완료 후 `alphazero_model_agent_latest.pth`로 자동 업그레이드됩니다.

### Q3. 왜 30일이 필요한가요?

**A:** 시계열 Feature(MA7, MA14, MA30, MACD 등)와 가격 추이 패턴 학습에
최소 30일의 연속 데이터가 필요합니다.
특히 MACD는 26일 이상 이력이 있어야 실값이 계산됩니다.

### Q4. SKIP 행동은 언제 나오나요?

**A:** SKIP은 가격 변화율 기반 레이블링 대상이 아닙니다.
정책 보정 단계에서 MCTS / ActionModel의 탐색 결과로 선택될 수 있으며,
안전 게이트(신뢰도 < 0.25 또는 엔트로피 > 1.58)가 발동하면 HOLD로 대체됩니다.

### Q5. Mac이 아니면 안 되나요?

**A:** 아니요. LaunchAgent는 macOS 전용이지만, 크론(cron)이나 systemd로 대체 가능합니다.

```bash
# Linux/Windows: 직접 cron 또는 Task Scheduler로 run_daily.py 실행
python3 crawlers/run_daily.py
```

MPS 가속은 Apple Silicon 전용이지만, CUDA/CPU로 자동 폴백됩니다.

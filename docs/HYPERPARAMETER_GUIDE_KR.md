# 하이퍼파라미터 설계 근거

GPU Advisor 프로젝트에서 사용된 모든 핵심 하이퍼파라미터와 그 선택 이유를 설명합니다.

## 1. 잠재 공간 차원: 256D

| 항목 | 값 |
|------|-----|
| 입력 차원 | 256D (GPU 시장 상태 벡터) |
| 잠재 공간 | 256D |
| 확장 비율 | 1:1 (입력=잠재) |

**입력 벡터 구성 (256D)**:

| 특징 그룹 | 차원 | 내용 |
|-----------|------|------|
| 가격 특징 | 60D | 정규화 가격, MA7/MA14/MA30, 변화율, 변동성 등 |
| 환율 특징 | 20D | USD/KRW, JPY/KRW, EUR/KRW 정규화 |
| 뉴스 특징 | 30D | 감성 점수, 기사 수, 긍정/부정 비율 |
| 시장 특징 | 20D | 판매자 수, 재고 상태 |
| 시간 특징 | 20D | 요일, 월, 연말 여부 |
| 기술 지표 | 106D | RSI, MACD, 모멘텀, 패딩 |

**왜 256인가?**

- **정보 용량**: 256D는 6개 카테고리 특징을 하나의 벡터로 압축하기에 충분한 차원입니다.
- **2의 거듭제곱**: GPU 텐서 연산에서 2^8 = 256은 메모리 정렬과 SIMD 연산에 최적화됩니다.
- **MuZero 논문 참고**: 원본 MuZero는 바둑 19×19 보드(361개 교차점)에 대해 256채널을 사용했습니다.
- **128 vs 256 vs 512 비교**:
  - 128D: 6개 특징 카테고리를 구분하기에 병목이 될 수 있음
  - 256D: 카테고리당 ~42D 할당 가능, 적절한 표현력
  - 512D: 파라미터가 2배로 증가하지만 과적합 위험

## 2. MCTS 시뮬레이션 횟수: 50회

| 항목 | 값 |
|------|-----|
| 시뮬레이션 수 | 50 |
| 행동 수 | 5 |
| 행동당 평균 탐색 | 10회 |

**왜 50인가?**

- **행동 공간 대비**: AlphaGo는 362개 행동에 1,600회 시뮬레이션을 사용합니다 (행동당 ~4.4회). 우리는 5개 행동이므로, 50회면 행동당 10회로 **AlphaGo보다 풍부한 탐색**입니다.
- **추론 속도**: 50회 × 5단계 롤아웃 = 250회 신경망 호출. CPU에서 약 0.5초로, 실시간 API 응답에 적합합니다.
- **수확체감**: 시뮬레이션 횟수가 증가하면 정확도가 올라가지만, 5개 행동에서는 30회 이상부터 수렴하기 시작합니다.

```
시뮬레이션 수 vs 추정 성능:
10회  → 행동당 2회, 탐색 불충분
30회  → 행동당 6회, 기본적 탐색
50회  → 행동당 10회, 안정적 탐색 ← 선택
100회 → 행동당 20회, 추론 시간 2배, 개선 미미
```

## 3. 행동 공간: 5개 행동

| 행동 | 의미 | 설계 근거 |
|------|------|-----------|
| BUY_NOW (0) | 즉시 구매 | 가격 하락 감지 시 매수 |
| WAIT_SHORT (1) | 단기 대기 (1주) | 소폭 하락 예상 |
| WAIT_LONG (2) | 중기 대기 (1개월) | 큰 하락 예상 |
| HOLD (3) | 관망 | 불확실성 높을 때 |
| SKIP (4) | 이번 사이클 패스 | 구매 자체 회피 |

**행동 레이블링 임계값** (`_action_from_delta()`):

| 가격 변화율 | 레이블 |
|------------|--------|
| `pct >= +2%` | BUY_NOW (0) |
| `-0.5% > pct >= -2%` | WAIT_SHORT (1) |
| `pct <= -2%` | WAIT_LONG (2) |
| 나머지 | HOLD (3) |

> **주의**: SKIP(4)은 평가기 보상 함수와 역방향 관계이므로 가격 데이터에서 레이블링하지 않습니다.
> 가격 하락(-6%)에서 WAIT_LONG 보상(+0.06) > SKIP 보상(-0.009)이므로 SKIP 레이블은 역학습을 유발합니다.

**왜 5개인가?**

- **3개 (매수/대기/관망)**: 너무 단순. "얼마나 대기할지"를 표현하지 못함
- **5개**: 시간 축(즉시/단기/중기)과 강도 축(매수/대기/회피)을 모두 커버
- **7개 이상**: GPU 구매 결정에서 더 세분화할 실질적 차이가 없음

## 4. 탐험 상수 (PUCT): c = √2 ≈ 1.414

MCTS는 AlphaZero 스타일의 PUCT(Predictor + UCT) 공식을 사용합니다:

```
UCB(s, a) = Q(s, a) + c × P(s, a) × √N_parent / (1 + N_child)
```

- `Q(s, a)`: 행동 가치 추정 (방문 평균 보상)
- `P(s, a)`: 예측 네트워크 f의 사전 확률
- `N_parent`: 부모 노드 방문 횟수
- `N_child`: 현재 노드 방문 횟수

**왜 √2인가?**

- **UCB1 이론**: Auer et al. (2002)이 증명한 최적 탐험 상수가 √2입니다.
- **탐험-활용 균형**: c가 작으면 활용(exploitation) 편향, c가 크면 탐험(exploration) 편향
  - c = 1.0: 탐험 부족, 초기 좋은 행동에 고착
  - c = √2: 이론적 최적, 모든 행동을 충분히 탐색
  - c = 2.0: 과도한 탐험, 수렴 느림
- **AlphaGo**: PUCT 변형을 사용하며 c_puct = 1.5~2.5 범위. 우리 값은 이 범위 내입니다.

## 5. 디리클레 노이즈 (루트 탐색 다양성)

| 항목 | 값 |
|------|-----|
| epsilon (ε) | 0.25 |
| alpha (α) | 0.03 |
| 적용 위치 | 루트 노드 사전 확률에만 |

```python
noise = Dirichlet([α] * action_dim)
policy_probs = (1 - ε) * policy_probs + ε * noise
```

**설계 근거**:
- AlphaZero 논문과 동일한 방식으로 루트 노드에만 노이즈를 추가하여 탐색 다양성 확보
- ε=0.25: 원래 사전 확률을 75% 유지하면서 노이즈 25% 혼합
- α=0.03: 낮은 값 = 한 행동에 집중되는 스파이크 노이즈 (탐색 강도를 높임)

## 6. 롤아웃 깊이: 5단계

| 항목 | 값 |
|------|-----|
| 롤아웃 깊이 | 5 |
| 할인율 | 0.99 |
| 5단계 후 유효 할인 | 0.99^5 ≈ 0.951 |

**왜 5단계인가?**

- **시간 해석**: 각 단계 = 1일. 5단계 = 향후 5일 시뮬레이션.
- **GPU 가격 특성**: 가격 변동의 주요 패턴은 주간(5영업일) 단위로 나타남
- **오차 누적**: Dynamics Network의 예측 오차는 단계마다 누적됩니다. 5단계 이후 예측 신뢰도가 급격히 감소
- **계산 비용**: 롤아웃 깊이 × 시뮬레이션 수 = 총 신경망 호출. 5 × 50 = 250회가 실시간 추론에 적합

## 7. 신경망 아키텍처

### 월드 모델 3개 네트워크 (h, g, f)

```
입력 256D → 은닉 512D (×4 잔차 블록) → 출력 256D/5D/1D
```

- **잔차 연결**: 모든 블록에 `x = x + block(x)` 적용 → 깊은 네트워크 학습 안정성
- **2배 확장**: 입력 256D의 2배인 512D는 충분한 비선형 표현력을 제공
- **파라미터 예산**: 총 ~18.9M 파라미터를 3개 네트워크에 분배

| 네트워크 | 역할 | 파라미터 |
|----------|------|----------|
| h (표현) | 256D 입력 → 256D 잠재 상태 | ~6M |
| g (역학) | 잠재 상태 + 행동 → 다음 상태 + 보상(μ, σ²) | ~6M |
| f (예측) | 잠재 상태 → 정책 + 가치 | ~6M |
| a (행동 모델) | 행동 임베딩 + 사전 확률 네트워크 | ~43K |

### ActionModel (a)

```
ActionEmbeddingLayer(num_actions=5, embed_dim=16)
ActionPriorNetwork: 256 → 128 → 64 → 5
총 파라미터: ~43K
```

- 잠재 상태에서 행동 사전 분포를 예측
- 4신호 보정 블렌드에서 15% 가중치로 활용
- `utility_bias` 하드코딩 휴리스틱을 데이터 기반 학습으로 대체

### 역학 네트워크의 보상 불확실성 헤드

```
g(s, a) → (next_state, reward_mean μ, reward_logvar)
σ² = exp(reward_logvar)    # 분산
σ  = reward_var.sqrt()     # 표준편차
```

## 8. 학습 하이퍼파라미터

| 항목 | 값 | 근거 |
|------|-----|------|
| 학습률 | 1e-4 | AdamW의 안정적 수렴 범위 |
| 배치 크기 | 32 | 일일 24개 GPU 모델 × 데이터 일수에 대한 효율적 샘플링 |
| 학습 스텝 | 500 | 데이터 크기 대비 과적합 방지 |
| 가중치 감쇠 | 1e-5 | L2 정규화로 과적합 억제 |
| 기울기 클리핑 | 1.0 | `clip_grad_norm_()` 반환값 캡처하여 실제 적용 |
| 옵티마이저 | AdamW | 가중치 감쇠 분리 적용 |

### 학습률 1e-4 근거

```
1e-3: 학습 불안정, 손실 진동
1e-4: 안정적 수렴, 500 스텝 내 수렴 ← 선택
1e-5: 수렴 너무 느림, 5000+ 스텝 필요
```

## 9. 손실 함수 가중치

총 손실 = 아래 5개 손실의 가중 합:

```python
total_loss = (
    1.0 * latent_loss          # 잠재 상태 일관성 (월드모델 핵심)
  + 1.0 * policy_loss          # 정책 KL 손실
  + 1.0 * value_loss           # 가치 MSE 손실
  + 0.5 * reward_nll_loss      # Gaussian NLL (불확실성 헤드 학습)
  + 0.3 * action_prior_loss    # ActionModel 지도 학습
)
```

**Gaussian NLL 공식**:
```
NLL = 0.5 × ((r - μ)² × exp(-logvar) + logvar)
```
- `logvar` 헤드에 실제 그래디언트를 전달하여 보상 불확실성 학습

**action_prior_loss**: CrossEntropy — ActionModel이 가격 데이터 레이블과 일치하도록 학습

## 10. 정책 보정 가중치 (4-신호 블렌드)

> **2026-04-03 업데이트**: MCTS 가중치 0.45→0.60 상향 (계획 결과 우선 강화)

```python
# 현재 (v3.1, 2026-04-03~)
calibrated = 0.60 × MCTS정책 + 0.20 × 보상정책 + 0.10 × f-net사전확률 + 0.10 × ActionModel사전확률

# 이전 (v3.0, ~2026-04-02)
# calibrated = 0.45 × MCTS정책 + 0.25 × 보상정책 + 0.15 × f-net사전확률 + 0.15 × ActionModel사전확률
```

| 출처 | 가중치 | 역할 |
|------|--------|------|
| MCTS 정책 | **0.60** | 핵심 의사결정 (계획 기반) — 상향 조정 |
| 보상 정책 | **0.20** | 기대 보상 기반 보정 |
| f-net 사전 확률 | **0.10** | 데이터 분포 기반 정규화 |
| ActionModel 사전 확률 | **0.10** | 학습된 행동 사전 확률 (utility_bias 대체) |

**왜 MCTS를 0.60으로 상향했는가?**

- 0.45에서 MCTS의 계획 결과가 보상/사전 신호에 희석되는 현상 관찰
- 0.60으로 계획 네트워크 결과를 더 직접적으로 반영하도록 튜닝
- 합계가 1.0이 되어 확률 분포를 보장

## 11. 엔트로피 붕괴 방지 (Anti-Collapse)

> **2026-04-03 업데이트**: 임계값 0.65→0.45로 낮춤 (더 자신감 있는 결정 허용)

```python
# 현재 (v3.1, 2026-04-03~)
min_entropy_target = 0.45  # 비트 (이전: 0.65)

if entropy(calibrated) < min_entropy_target:
    alpha = min(0.55, (min_entropy_target - entropy_now) / max(min_entropy_target, 1e-6))
    uniform = ones(num_actions) / num_actions   # 균등 분포 (편향 없음)
    calibrated = (1 - alpha) * calibrated + alpha * uniform
```

**설계 근거**:
- **균등 분포 사용**: 경험적 사전 확률(예: HOLD 60%)은 훈련 편향을 증폭시키므로 사용하지 않음
- `alpha` 상한 0.55: 균등화가 너무 강해지는 것을 방지
- `min_entropy_target = 0.45` (변경): 신뢰도 높은 결정을 Anti-collapse가 희석시키지 않도록 임계 완화. 이론적 최대 신뢰도가 0.65에서 ~0.80으로 상승.

## 13. 할인율: γ = 0.99

- **의미**: 미래 보상의 현재 가치 비율
- **0.99 선택 이유**: GPU 가격 결정은 장기적 관점이 중요. 5일 후 보상도 현재의 95.1% 가치로 반영
- **0.9 vs 0.99 vs 0.999**:
  - 0.9: 1일 후 보상만 90% 반영, 너무 근시안적
  - 0.99: 5일 후 95.1%, 30일 후 74.0% — 적절한 장기 관점
  - 0.999: 30일 후 97.0% — 장기 보상을 과대평가
